Linux Upskill Challenge: Day 2

  • 19/9/2022

blog ini merupakan bagian dari linux upskill challenge series. berisi catatan proses belajar saya dalam mengikuti panduan Linux Upskill Chalenge

Sekarang ini sebagian besar desktop komputer contohnya Windows, Mac OS dan GNU/Linux modern saat ini, sudah meninggalkan CLI dan menggantinya dengan GUI. namun untuk menjadi sysadmin linux hal tersebut tidak berlaku, kita akan sering menemui masalah atau bug yang tidak akan dapat diselesaikan melalui GUI. Dan masih sangat penting sekarang ini untuk mempelajari hal tersebut.

Dalam hal yang kaitannya dengan produktifitas tentu CLI lebih banyak dipilih, dikarenakan CLI sangat powerful dan efektif dalam mendukung alur kerja kita. walaupun kurva belajar dalam CLI sangat curam namun itu sepadan dengan hasil yang didapatkan.

Kembali ke topik, dalam sistem file komputer tidak hanya linux, ada yang disebut dengan directory hierarchy atau lebih mudahnya struktur folder. dalam Windows contonya kita menemukan file pictures, desktop, video dll, yang jika kita runut keatasnya file-file tersebut merupakan bagian dari direktori Steve dan kita lihat lagi ke atasnya kita menemukan bahwa folder Steve merupakan bagian dari folder Steve dan diatasnya lagi ada C: folder, maka jika kita buat jalur maka akan seperti ini C:/Users/Steve/Destop, dan itu disebut dengan “path”. lihat https://www.linuxfoundation.org/blog/blog/classic-sysadmin-the-linux-filesystem-explained

Path di setiap sistem operasi berbeda-beda, dalam unix-like os path dimulai dari “root” yang disimbolkan dengan / dan dari sinilah semua direktori linux dicabangkan. kita bisa mengatur, mengorganisir, dan memonitor pohon/tree direktori dengan file manager seperti nemo, nautillus, pcmanfm dan lain-lain, semua aplikasi yang saya sebutkan tadi berbentuk GUI aplikasi, sehingga kita tidak bisa mengaksesnya melalui terminal.

Dan hari ini kita akan mempelajari bagaimana menavigasi direktori linux, dengan menggunakan perintah shell.
